# Contributing

Thanks for contributing to d-dash.

## Development setup

1. Install Node.js 18+ and npm.
2. Run `npm install` in the repository root.
3. Build all packages with `npm run build`.
4. Run tests with `npm run test`.

## Workflow

1. Keep changes small and focused.
2. Add tests for business logic changes.
3. Update docs in the same change when behavior changes.
4. Follow rules in `docs/CODE_RULES.md` and `docs/CONTRACTS.md`.

## Pull requests

1. Include a clear summary and motivation.
2. Include test evidence for behavior changes.
3. Mention migration impact for stable contract changes.
4. Keep public API and docs aligned.
